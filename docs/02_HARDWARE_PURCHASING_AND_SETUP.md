# 02. 手机硬件、外设采购与配置

## 模块目标

硬件模块的目标是建立稳定、可重复、低噪声的物理链路：

```text
PC USB <----> A 端安卓手机
PC 音频输出 ----> A 端手机麦克风输入
A 端手机耳机输出 ----> PC 音频输入
```

这个模块不负责微信业务，也不负责 AI 推理。它只证明设备可靠、声卡可用、音频链路干净、长期运行稳定。

## 采购原则

默认条件：

- 中国大陆购买环境。
- 复用旧安卓手机。
- 稳定优先，不追求最低价。
- 不 root。
- 通过真实 USB、真实声卡、真实线材工作。

优先级：

1. 稳定识别。
2. 低底噪。
3. 线材牢固。
4. 长期插电不过热。
5. 采购容易替换。

## 采购清单

### A 端安卓手机

复用旧手机的建议规格：

- Android 10 或更高。
- USB 调试可开启。
- 电池不过度鼓包。
- 能关闭锁屏密码或设置简单解锁路径。
- 能关闭省电限制。
- 能长期亮屏或保持后台稳定。
- 微信版本可正常登录。

不推荐：

- 系统过旧，无法稳定运行当前微信。
- USB 口接触不良。
- 充电发热严重。
- 厂商系统频繁杀后台且无法关闭。
- 屏幕严重破损，影响 UI 自动化定位。

验收标准：

- `adb devices` 显示 `device`。
- 连续 30 分钟 ADB 不断开。
- 能截图。
- 能按电源键、Home/返回键。
- 能保持微信前台。

### USB 数据与供电

建议：

- 使用质量稳定的数据线，确认支持数据传输，不只是充电线。
- 手机连接 PC 用数据线，供电不足时可使用带供电的 USB Hub。
- 避免过长线材。
- 线材和接口固定，防止早晨自动运行时松动。

验收标准：

- 插拔 5 次后 ADB 都能恢复。
- 长时间连接不反复弹出授权。
- Windows 设备管理器无频繁断连提示。

### 外置声卡

最低能力：

- Windows 免驱或驱动稳定。
- 至少 1 路输入、1 路输出。
- 输入输出能同时工作。
- 支持 16kHz 或 48kHz。
- 底噪可控。

稳定优先选择：

- 入门 USB 音频接口。
- 独立音量旋钮。
- 输入输出接口清楚。
- Windows 设备名稳定。

可搜索关键词：

- `USB 外置声卡 线性输入 线性输出`
- `USB 音频接口 RCA 输入输出`
- `免驱 USB 声卡 麦克风 输入 耳机 输出`
- `入门 音频接口 直播 声卡 监听`

采购避坑：

- 只支持耳麦二合一、没有独立输入输出的极简声卡风险较高。
- 只面向娱乐变声的声卡可能驱动复杂。
- 有强制音效、降噪、混响且无法关闭的设备不适合。
- 声卡设备名会变化的产品不适合长期自动化。

验收标准：

- Windows 能看到独立输入和输出设备。
- Python 能枚举到设备。
- 输入输出可同时打开。
- 播放测试音时回录波形稳定。
- 静音底噪目标低于 `-50 dBFS`。

### 音频线材

需要准备：

- 3.5mm 公对公音频线。
- 手机 CTIA 耳麦转接头。
- 必要时准备 TRRS 分线器。
- 必要时准备衰减线，避免 PC 输出过大打爆手机麦克风输入。

关键概念：

- 普通 3.5mm TRS 线通常只传左右声道和地。
- 手机耳麦口通常是 TRRS，包含耳机输出和麦克风输入。
- Android 手机常见 CTIA 标准，但旧设备或特殊转接头可能有差异。
- CTIA/OMTP 不匹配会导致麦克风无输入或噪声异常。

验收标准：

- PC 播放声音，A 手机通话对端能听到。
- B 手机说话，PC 输入能录到。
- 没有明显单边失声。
- 没有持续电流声。

### 共地隔离器

作用：

- 降低 USB、PC、手机、声卡之间形成的地环路噪声。
- 减少嗡声和电流声。

建议：

- 准备至少一个 3.5mm 音频隔离器。
- 如果输入和输出链路都受影响，可能需要两个。
- 隔离器应放在噪声最明显的一段链路上测试。

验收标准：

- 加入隔离器后底噪下降。
- 语音可懂度不明显下降。
- 不引入明显失真。

### 固定与散热

建议：

- 手机支架固定 A 端手机。
- 线材贴标签。
- USB 与音频线分开走线，减少干扰。
- 手机不要被被褥、书本覆盖。
- 长期插电时观察温度。

验收标准：

- 连续 2 小时测试不掉线。
- 手机不过热。
- 屏幕不会自动锁死导致 UI 不可控。

## Windows 配置建议

### 声音控制面板

对输入设备：

- 禁用自动增益、增强、降噪、回声消除等系统音效。
- 设置固定采样率，例如 16kHz 或 48kHz。
- 关闭“允许应用独占控制”需要根据实际驱动测试。

对输出设备：

- 关闭响度均衡、空间音频、音效增强。
- 输出音量从较低值开始，例如 10% 到 20%。
- 逐步提高，避免手机麦克风输入削波。

验收标准：

- 设备名与 `config.yaml` 一致。
- 重启后设备名不变化。
- 默认通信设备变化不会影响配置指定设备。

### ADB 配置

步骤：

1. 手机开启开发者选项。
2. 开启 USB 调试。
3. 连接 PC。
4. 手机上确认 RSA 授权。
5. PC 执行 `adb devices`。

验收标准：

```text
List of devices attached
<device-id>    device
```

失败处理：

- `unauthorized`：重新插拔并在手机上确认授权。
- 无设备：检查线材、USB 模式、驱动。
- 多设备：配置必须指定 `android_device_id`，不允许默认随便选。

## 待实现小功能

### 1. ADB 健康检查

建议位置：

- `scripts/check_devices.py`
- 后续可抽象为 `HardwareProbePort`

功能：

- 检查 `adb` 是否存在。
- 列出设备。
- 确认只有一个目标设备，或配置指定设备存在。
- 查询屏幕分辨率。
- 查询电量和充电状态。
- 执行截图测试。
- 输出 JSON 或表格报告。

测试标准：

- 无 ADB 时输出清晰错误。
- 无设备时不抛未捕获异常。
- 多设备时要求配置指定目标。
- fake adb 输出可单元测试解析逻辑。

### 2. 音频设备枚举

建议位置：

- `scripts/list_audio_devices.py`
- `src/kaslana/adapters/audio/`

功能：

- 使用 `sounddevice.query_devices()` 枚举设备。
- 标记输入通道数、输出通道数。
- 标记默认采样率。
- 输出推荐写入配置的设备名或 index。

测试标准：

- 未安装 `sounddevice` 时给出安装提示。
- 设备列表为空时不崩溃。
- 配置中的设备名必须能匹配到枚举结果。

### 3. 音频回环校准

建议位置：

- `scripts/calibrate_audio.py`

功能：

- 播放 1kHz 测试音。
- 录制输入。
- 计算峰值、RMS、底噪、削波比例。
- 输出建议：提高音量、降低音量、检查线材、加入隔离器。

测试标准：

- 静音底噪低于 `-50 dBFS` 为合格。
- 测试音峰值在 `-12 dBFS` 到 `-3 dBFS` 为推荐区间。
- 削波样本比例接近 0。
- 录音文件可选保存到 ignored 目录，不提交。

### 4. 长时间稳定性测试

建议位置：

- `scripts/hardware_soak_test.py`

功能：

- 每分钟检查 ADB 连接。
- 每分钟短录音。
- 每 10 分钟截图。
- 持续 2 小时。
- 生成稳定性报告。

测试标准：

- 2 小时内 ADB 不断连。
- 音频设备不消失。
- 手机不过热。
- 截图和录音都能按计划完成。

## 常见失败模式

- ADB 授权反复丢失。
- 手机省电策略杀掉微信。
- 手机屏幕锁定导致 UI 不可控。
- USB Hub 供电不足。
- 声卡输入输出设备名变化。
- 手机麦克风输入过载导致削波。
- 地环路造成持续嗡声。
- CTIA/OMTP 转接不匹配。
- Windows 自动启用音效增强。

## 人工验收步骤

1. 连接 A 端手机和 PC。
2. 执行 `adb devices`，确认设备状态为 `device`。
3. 用 ADB 截图，确认截图保存成功。
4. 连接外置声卡。
5. 枚举音频设备，确认输入输出都存在。
6. 连接物理音频环回。
7. 播放测试音并回录。
8. 检查底噪和削波。
9. 连续运行 30 分钟短测。
10. 记录最终设备名、线材接法、音量位置和配置值。
