# 12. Xiaomi 13 硬件诊断个人操作手册

这份手册给项目本人使用。目标是把 Xiaomi 13 准备成稳定的 A 端安卓机，让后续 `scripts/check_devices.py` 能安全读取设备状态。

本阶段只做硬件诊断准备：

- 连接 USB。
- 开启 ADB。
- 确认手机授权。
- 调整锁屏、省电和后台设置。
- 准备外置声卡和线材。
- 验证 PC 能看到设备。

本阶段不打开微信、不搜索联系人、不发起语音通话。

## 0. 安全边界

不要做：

- 不 root。
- 不解锁 bootloader。
- 不刷机。
- 不安装微信协议机器人。
- 不使用微信非官方 API。
- 不 Hook 微信。
- 不修改微信内存。
- 不真实拨号。

要做：

- 使用真实 Xiaomi 13。
- 使用真实 USB 数据线。
- 只通过 ADB 做设备诊断。
- 所有真实设备 ID 只写进未提交的本地配置。
- 发现不确定弹窗时先停下来人工确认。

## 1. 准备物品

先准备：

- Xiaomi 13 作为 A 端安卓机。
- 一根确定支持数据传输的 USB-C 数据线。
- Windows PC。
- Android platform-tools 或其他可用 `adb`。
- 外置声卡，后续音频阶段使用。
- 手机支架，后续长时间测试使用。

可稍后准备：

- 3.5mm 音频线。
- USB-C 转 3.5mm CTIA 耳麦转接头。
- TRRS 分线器。
- 3.5mm 共地隔离器。
- 衰减线或可调音量的小型音频设备。

## 2. 开启 Xiaomi 13 开发者选项

在手机上操作：

1. 打开 `设置`。
2. 进入 `我的设备`。
3. 进入 `全部参数与信息`。
4. 连续点击 `MIUI 版本` 或 `OS 版本`。
5. 看到已进入开发者模式的提示后停止。

不同系统版本的入口名称可能略有差异。只要最终能在 `更多设置` 中看到 `开发者选项` 即可。

## 3. 开启 USB 调试

在手机上操作：

1. 打开 `设置`。
2. 进入 `更多设置`。
3. 进入 `开发者选项`。
4. 打开 `USB 调试`。

连接 PC 后，手机可能弹出 RSA 授权窗口：

1. 勾选 `始终允许使用这台计算机进行调试`。
2. 点击 `允许`。

如果后续 UI 自动化阶段出现无法点击、无法输入或权限不足，再考虑开启：

- `USB 调试（安全设置）`
- `通过 USB 安装`

第一阶段硬件诊断通常不强制需要这两项。

## 4. 锁屏与息屏设置

推荐测试期配置：

- 关闭锁屏密码，或改成滑动解锁。
- 将自动锁屏时间调到 10 分钟或更长。
- 如果系统允许，测试时设置为不自动锁屏。
- 保持手机有足够电量，并尽量插电测试。

如果不想关闭锁屏密码，第一阶段仍可做 ADB 连接诊断，但后续 UI 自动化会更容易卡在锁屏页。等进入微信自动化阶段前，最好再改成可被脚本稳定处理的锁屏方案。

## 5. 电源、省电与后台设置

在手机上操作：

- 关闭省电模式。
- 关闭超级省电模式。
- 允许微信后台运行。
- 不要启用会强制清理后台的安全策略。
- 长时间插 USB 时观察温度，不要盖住手机。

如果系统有应用省电设置，给微信选择更宽松的后台策略。第一阶段不会启动微信，但提前设置可以减少后续排查成本。

## 6. USB 连接步骤

1. 用数据线连接 Xiaomi 13 和 PC。
2. 手机弹出 USB 用途时，选择文件传输或数据传输模式。
3. 如果弹出 USB 调试授权，按第 3 节确认。
4. 在 PC PowerShell 中执行：

```powershell
adb devices
```

理想输出：

```text
List of devices attached
<device-id>    device
```

只要状态是 `device`，第一关就过了。

## 7. ADB 常见问题

### `unauthorized`

含义：PC 看到了手机，但手机还没授权调试。

处理：

- 看手机屏幕，确认 RSA 授权。
- 勾选始终允许。
- 重新执行 `adb devices`。
- 如果没有弹窗，拔插 USB 后重试。
- 必要时在开发者选项里撤销 USB 调试授权，再重新连接。

### 没有任何设备

处理：

- 确认数据线支持数据传输。
- 换 USB 口。
- 换线。
- 确认手机 USB 用途不是仅充电。
- 确认 ADB 已安装并在 PATH 中，或使用完整 `adb.exe` 路径。
- 检查 Windows 设备管理器是否有异常 Android 设备。

### 多个设备

处理：

- 暂时只连接 Xiaomi 13。
- 后续在本地 `config/config.yaml` 中明确填写目标 `android_device_id`。
- 不要把真实设备 ID 提交进 Git。

### 状态反复断开

处理：

- 换更短、更稳定的数据线。
- 避免 USB Hub，或使用带供电的 Hub。
- 检查手机接口松动。
- 观察手机是否过热。
- 连续测试 30 分钟，确认 `adb devices` 状态稳定。

## 8. 手工诊断命令

这些命令只读设备状态，不打开微信、不拨号。

查看设备列表：

```powershell
adb devices -l
```

读取屏幕分辨率：

```powershell
adb shell wm size
```

读取电池状态：

```powershell
adb shell dumpsys battery
```

保存一张截图到 PC：

```powershell
adb exec-out screencap -p > diagnostics\manual_screen.png
```

如果 `diagnostics` 目录不存在，可先创建：

```powershell
New-Item -ItemType Directory -Force diagnostics
```

`diagnostics/` 应只保存本地诊断产物，不提交进 Git。

## 9. 外置声卡准备

第一阶段可以先不接完整音频回环，但建议提前完成这些准备：

- 插入外置声卡。
- 在 Windows 声音设置中确认能看到输入设备和输出设备。
- 关闭输入增强、降噪、回声消除、自动增益等效果。
- 关闭输出空间音频、响度均衡和音效增强。
- 输出音量从 10% 到 20% 开始。
- 记下设备显示名称，后续写入本地配置。

不要急着把 PC 输出直接打进手机麦克风输入。音频阶段会单独做底噪和削波校准。

## 10. 第一阶段验收清单

完成后应满足：

- Xiaomi 13 已开启开发者选项。
- `USB 调试` 已开启。
- PC 已获得 RSA 授权。
- `adb devices` 显示状态为 `device`。
- 能执行 `adb shell wm size`。
- 能执行 `adb shell dumpsys battery`。
- 能保存一张截图。
- 锁屏和息屏设置不会阻塞短时间诊断。
- 省电策略不会频繁断开 ADB。
- 连续连接 30 分钟没有明显断连。

达到这些条件后，就可以让后续 AI 助理执行 `docs/13_AI_TASK_HARDWARE_DIAGNOSTICS.md` 中的硬件诊断开发任务。
