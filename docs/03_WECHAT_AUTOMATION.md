# 03. 微信自动呼叫模块

## 模块目标

微信自动呼叫模块负责在 A 端安卓手机上模拟人类操作：

1. 唤醒设备。
2. 打开微信。
3. 搜索本人 B 机联系人。
4. 发起语音通话。
5. 等待接通。
6. 在失败或结束时挂断。

这个模块只做真实 UI 自动化，不碰微信协议。

## 绝对安全边界

允许：

- ADB。
- `uiautomator2`。
- 截图。
- UI 层级 XML。
- 控件点击。
- 文本输入。
- 按键模拟。
- 坐标点击兜底。

禁止：

- 微信协议逆向。
- Hook。
- 内存读取或修改。
- 非官方微信 API。
- 伪造微信网络请求。
- 使用第三方“微信机器人协议”。

如果后续 AI 助手建议使用协议登录、token、wxid 消息接口或内存注入，应立即拒绝。

## 当前代码位置

- 抽象接口：`src/kaslana/ports/automation.py`
- 占位实现：`src/kaslana/adapters/automation/uiautomator2_wechat.py`
- 配置示例：`config/config.example.yaml`

当前 `AutomationPort` 方法：

- `wake_device()`
- `open_wechat()`
- `dial_voice_call(contact_alias)`
- `wait_for_call_connected(timeout_s)`
- `hang_up()`

## 输入与输出

输入：

- `android_device_id`
- 微信包名：`com.tencent.mm`
- 目标联系人 alias
- UI 坐标兜底值
- 超时时间
- 是否允许真实拨号

输出：

- 自动化步骤成功或失败。
- 接通状态。
- 截图和 UI XML 诊断。
- 失败原因。

建议后续增加结构：

```text
AutomationStepResult
  step_name
  ok
  elapsed_ms
  screenshot_path
  hierarchy_path
  reason
```

## 定位策略

默认策略：控件/文本/XML 层级优先，坐标兜底。

优先级：

1. 使用稳定文本定位，例如“搜索”“语音通话”“挂断”。
2. 使用控件层级和 class name 辅助定位。
3. 使用截图人工校准的坐标兜底。
4. 任何兜底坐标都必须来自配置。

开发建议：

- 不要把坐标写在 Python 代码里。
- 每次点击前尽量确认当前页面。
- 每次失败保存截图和 XML。
- 对微信版本变化保持宽容，避免单一文本硬匹配。

## 分级解锁测试

真实拨号风险高，因此必须分级测试。

### Level 1：设备连接与截图

功能：

- 连接指定 ADB 设备。
- 获取屏幕分辨率。
- 截图。
- 获取 UI 层级。

禁止：

- 打开微信。
- 点击微信内部控件。

测试标准：

- 设备 ID 与配置一致。
- 截图非空。
- UI XML 非空。
- 失败时给出 ADB 诊断。

### Level 2：唤醒与打开微信

功能：

- 唤醒手机。
- 解锁或保持亮屏。
- 打开微信包名。
- 等待微信首页可见。

禁止：

- 搜索联系人。
- 拨打通话。

测试标准：

- 能从黑屏进入可操作状态。
- 微信在前台。
- 未登录或弹窗遮挡时保守失败。

### Level 3：搜索联系人

功能：

- 点击搜索。
- 输入本人 B 机联系人 alias。
- 定位搜索结果。

禁止：

- 点击语音通话按钮。

测试标准：

- 只找到一个明确目标。
- 如果多个候选，必须失败并要求人工处理。
- 未找到联系人时失败退出。

### Level 4：短拨号测试

功能：

- 点击目标联系人。
- 点击语音通话。
- 等待通话界面。
- 立即挂断或由你手动接听后挂断。

前置条件：

- 配置显式允许真实拨号。
- 你在现场观察。
- B 端手机可用。

测试标准：

- 不误拨其他联系人。
- 能在 5 秒内执行挂断。
- 失败路径也会尝试挂断。

### Level 5：接入完整通话链路

功能：

- 自动拨号。
- 接通检测。
- 播放问候。
- 后续交给音频和 AI 链路。

测试标准：

- Level 1-4 全部通过。
- 声卡和音频校准通过。
- TTS 和 LLM 健康检查通过。
- `preflight` 通过。

## 待实现小功能

### 1. `uiautomator2` 设备连接

开发建议：

- 从配置读取 `android_device_id`。
- 如果配置为空且只有一个设备，可提示但不建议自动选择。
- 连接后立即执行截图和层级读取。

测试标准：

- fake uiautomator client 可测试连接逻辑。
- 无设备、未授权、多设备都有清晰错误。

### 2. 唤醒与解锁

开发建议：

- 先支持无锁屏或已解锁设备。
- 后续再支持配置化滑动解锁。
- 唤醒后检查屏幕是否亮起。

测试标准：

- 黑屏状态可唤醒。
- 已亮屏状态重复执行不出错。
- 无法解锁时不继续打开微信。

### 3. 打开微信

开发建议：

- 使用包名启动。
- 等待首页关键 UI 出现。
- 处理权限弹窗、升级弹窗、网络异常弹窗。

测试标准：

- 微信已在前台时重复执行无害。
- 微信未登录时返回明确失败。
- 启动超时不继续后续步骤。

### 4. 搜索联系人

开发建议：

- 文本输入前清空搜索框。
- 输入 alias 后等待结果稳定。
- 结果匹配必须保守，宁可失败，不要点错。

测试标准：

- 未找到联系人失败。
- 多个相似联系人失败。
- 搜索结果 XML fixture 可离线测试。

### 5. 发起语音通话

开发建议：

- 引入配置 `allow_real_call`，默认 false。
- 未开启时走 dry-run，只记录即将点击的控件。
- 点击前保存截图。

测试标准：

- 默认配置不会真实拨号。
- 开启真实拨号必须有显式配置。
- 点击后能识别通话界面。

### 6. 接通检测

开发建议：

- 第一版根据 UI 文案、按钮状态、通话计时判断。
- 第二版可结合 VAD 检测 B 端声音。
- 超时必须挂断。

测试标准：

- 对方未接听进入超时。
- 对方接听进入 `GREETING`。
- 对方拒接或忙线进入 `HUNG_UP`。

### 7. 挂断

开发建议：

- 挂断操作必须幂等。
- 当前不在通话界面时不能乱点。
- 失败时至少返回桌面或微信安全页。

测试标准：

- 正常通话可挂断。
- 未接通等待页可取消。
- 已挂断时再次调用不产生危险点击。

## 常见失败模式

- 手机未解锁。
- 微信升级弹窗遮挡。
- 微信未登录。
- 搜索结果为空。
- 同名联系人导致误匹配风险。
- UI 文案因微信版本变化失效。
- 屏幕分辨率不同导致坐标兜底失效。
- B 端未接听或拒接。
- 通话界面按钮位置变化。

## 人工验收步骤

1. 连接 A 端手机，确认 `adb devices`。
2. 执行 Level 1，保存截图和 XML。
3. 执行 Level 2，确认微信首页。
4. 执行 Level 3，确认搜索到本人 B 机联系人。
5. 人工核对联系人头像和备注。
6. 开启 `allow_real_call`。
7. 执行 Level 4，观察 B 机收到通话。
8. 验证挂断。
9. 记录微信版本、手机分辨率、关键截图。

## 验收门槛

微信自动化模块合入完整调度前，必须满足：

- 单元测试覆盖选择器解析。
- 真机 Level 1-4 通过。
- 失败路径都不会继续点击危险按钮。
- 默认配置绝不真实拨号。
- 所有真实拨号都需要显式配置开启。
